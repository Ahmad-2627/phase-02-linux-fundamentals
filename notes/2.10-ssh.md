# 🔑 Topic 2.10 — SSH: Remote Access

## Picture this
You're sitting in Lahore. A server you need to test is sitting in a data center somewhere else entirely. SSH is the secure tunnel that lets you sit at your keyboard and fully control that far-away machine like you're right in front of it.

## 1–2. Concept
**SSH (Secure Shell)** = a protocol (a set of rules for communication) that lets you securely log into and control a remote computer over a network.

"Secure" matters — SSH **encrypts** everything (scrambles data so no one else can read it) between your computer and the remote one.

| Command | Meaning |
|---|---|
| `ssh user@ip` | Connect to a remote machine |
| `ssh-keygen` | Generate a key pair for login without password |
| `scp file user@ip:/path` | Securely copy a file to a remote machine |

## 3. Internal Working
SSH uses **asymmetric encryption** (two related keys — a public one you share, a private one you keep secret) to first verify identity, then switches to fast encrypted communication for the actual session. This is why SSH is trusted over old tools like Telnet, which sent everything in plain, readable text.

## 4. Why It Matters
SSH is how you'll access almost every Linux server during a pentest — and how you'll manage your own Kali VM remotely later. It's also a common **target** — weak SSH passwords are a top entry point in real attacks.

## 5. Real-World Example
A pentester finds SSH open on a target server (port 22). They try common/weak passwords (with authorization) using a tool like Hydra — if successful, they get a full remote shell (control) on that machine.

## 6. Attacker Perspective
Attackers scan for open SSH ports, then attempt password guessing (**brute-forcing**) or search for leaked SSH keys — a stolen private key equals instant access, no password needed.

## 7. Defender Perspective
Defenders disable password-based SSH login entirely, using only key-based authentication, and often restrict which IPs can even attempt to connect.

## 8. Common Mistakes
- Confusing SSH (secure) with Telnet (old, unencrypted, insecure — avoid).
- Not protecting your private key file — if leaked, it's like handing over your house key.

## 9. Practical Lab
1. `sudo systemctl start ssh` → start SSH service on Kali
2. `ssh kali@127.0.0.1` → connect to yourself (localhost) as a test
3. Type `yes` to trust the connection, enter password
4. You're now "remotely" inside your own machine — type `whoami`, `exit` to leave

## 10. Key Terms
| Term | Meaning |
|---|---|
| SSH | Secure Shell — encrypted remote access protocol |
| Encryption | Scrambling data so only authorized parties can read it |
| Public/Private key | A matched pair used to verify identity securely |
| Brute-force | Repeatedly guessing passwords until one works |
