# 📁 Topic 2.5 — File and Directory Management

## Picture this
You've learned to walk around the building (navigation). Now it's time to actually build rooms, move furniture, and throw out trash — creating, copying, moving, and deleting files and folders.

## 1–2. Concept
| Command | Meaning |
|---|---|
| `mkdir name` | Make Directory — create a new folder |
| `touch file.txt` | Create a new empty file |
| `cp source dest` | Copy a file/folder |
| `mv source dest` | Move (or rename) a file/folder |
| `rm file` | Remove (delete) a file |
| `rm -r folder` | Remove a folder and everything inside it |
| `cat file` | Display file contents on screen |

## 3. Internal Working
Linux doesn't have a "Recycle Bin" by default in the terminal — `rm` deletes immediately and permanently. `mv` doesn't actually copy data; it just updates the file's location pointer (that's why moving is instant even for huge files, but copying across drives takes time).

## 4. Why It Matters
You'll constantly create folders for tools/loot (data grabbed from targets), copy exploit scripts, move files between directories, and clean up after labs. This is daily-use stuff.

## 5. Real-World Example
After exploiting a machine, a pentester often finds a password file and uses `cp` to copy it to their own analysis folder before cracking it — never modifying the original evidence.

## 6. Attacker Perspective
Attackers use `rm` to delete their tracks (log files, temp tools) after an operation — this is called **anti-forensics** (hiding evidence of what they did).

## 7. Defender Perspective
Defenders watch for suspicious `rm` activity on log files — deleted logs are a huge red flag during incident investigation.

## 8. Common Mistakes
- Running `rm -r` carelessly — it deletes with **no confirmation and no undo**. Always double-check the path first with `pwd`/`ls`.
- Forgetting `-r` when deleting a folder (plain `rm` only works on files).

## 9. Practical Lab
1. `mkdir test-lab` → `cd test-lab`
2. `touch notes.txt`
3. `echo "My first Linux file" > notes.txt`
4. `cat notes.txt` → confirm text appears
5. `cp notes.txt backup.txt` → `ls` (see both files)
6. `mv backup.txt renamed.txt` → `ls`
7. `rm renamed.txt` → `ls` (confirm it's gone)

**Key learning point:** There's no "Are you sure?" in Linux — the terminal trusts you completely.

## 10. Key Terms
| Term | Meaning |
|---|---|
| Directory | Folder |
| `stdout` | Output printed to your screen |
| `>` | Redirects output into a file (overwrites) |
