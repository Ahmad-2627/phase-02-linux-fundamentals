# 👥 Topic 2.7 — Users and Groups

## Picture this
Every office building has employees, managers, and visitors — each with different access cards. Linux works the same way: every account belongs to a system of users and groups that decides who can do what.

## 1–2. Concept
- **User** = an individual account on the system (e.g., `kali`, `root`)
- **root** = the "superuser" — has unlimited power over the entire system (like a master key)
- **Group** = a collection of users who share the same permissions (e.g., "admin" group)
- Every user has a **UID** (User ID number) and belongs to at least one group (**GID** — Group ID)

Key files:
- `/etc/passwd` → list of all users
- `/etc/group` → list of all groups
- `/etc/shadow` → stores password hashes (encrypted passwords), readable only by root

## 3. Internal Working
When you log in, Linux checks your username against `/etc/passwd`, verifies your password hash against `/etc/shadow`, then loads your UID/GID — this determines what files and commands you're allowed to touch.

## 4. Why It Matters
Privilege escalation (moving from a low-level user to root) is one of the most important skills in pentesting. You can't understand escalation without understanding the user/group system first.

## 5. Real-World Example
A web application runs as a low-privilege user called `www-data`. If a pentester exploits the app, they only get `www-data` access — not root. The real challenge (privilege escalation) is figuring out how to become root from there.

## 6. Attacker Perspective
Attackers check `/etc/passwd` and current group memberships (`groups` command) immediately after gaining access — this tells them what they can and can't do yet.

## 7. Defender Perspective
Defenders follow the **principle of least privilege** — giving each user/service only the minimum access it needs, so even if compromised, damage stays limited.

## 8. Common Mistakes
- Confusing "root" (the superuser account) with "sudo" (a command that temporarily grants root-level power to a normal user).
- Not checking group memberships — sometimes a "normal" user is secretly in a powerful group.

## 9. Practical Lab
1. `whoami` → confirm current user
2. `id` → see your UID, GID, and groups
3. `cat /etc/passwd` → view all users (don't worry if it looks messy)
4. `groups` → see which groups you belong to
5. `sudo whoami` → enter password, confirm output shows `root`

## 10. Key Terms
| Term | Meaning |
|---|---|
| root | The superuser with full system control |
| sudo | "Superuser do" — temporarily run a command as root |
| UID/GID | Numeric ID for a user/group |
| Least privilege | Giving only the minimum access necessary |
