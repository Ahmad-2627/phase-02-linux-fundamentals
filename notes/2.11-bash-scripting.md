# 📜 Topic 2.11 — Bash Scripting Basics

## Picture this
Imagine typing the same 10 commands every single day to set up your lab. Boring, slow, error-prone. A **Bash script** is you writing those 10 commands once into a file — then running that file forever after with one word.

## 1–2. Concept
**Bash script** = a text file full of terminal commands that run automatically, top to bottom, when executed.

Basic structure:
```bash
#!/bin/bash
echo "Starting scan..."
```
- `#!/bin/bash` = the **shebang** — tells Linux "run this file using Bash"
- Every line after runs as if you typed it manually

Key building blocks:
| Concept | Example | Meaning |
|---|---|---|
| Variable | `name="Ahmad"` | Stores a value to reuse |
| If statement | `if [ condition ]; then...fi` | Runs code only if something is true |
| Loop | `for i in 1 2 3; do...done` | Repeats an action |

## 3. Internal Working
When you run a script, Bash reads it line by line, treating each line exactly like a typed command. Variables and loops just let you avoid repeating yourself.

## 4. Why It Matters
Pentesters automate repetitive tasks — scanning many IPs, checking many ports, organizing output files — with small Bash scripts instead of doing it by hand each time.

## 5. Real-World Example
A pentester writes a 5-line script that scans a list of 50 target IPs with Nmap automatically overnight, instead of running 50 manual commands.

## 6. Attacker Perspective
Attackers automate reconnaissance and exploitation steps with scripts to move fast and reduce human error during an operation.

## 7. Defender Perspective
Defenders write scripts too — automated log monitoring and alerting scripts that flag suspicious activity instantly instead of waiting for a human to notice.

## 8. Common Mistakes
- Forgetting to make the script executable (`chmod +x script.sh`) before running it.
- Forgetting the shebang line — script may run with the wrong interpreter.

## 9. Practical Lab
1. `nano myscript.sh` (opens a text editor)
2. Type:
```bash
#!/bin/bash
echo "Hello, Ahmad. This is your first script!"
name="Pentester"
echo "You are becoming a $name"
```
3. Save and exit (`Ctrl+O`, Enter, `Ctrl+X`)
4. `chmod +x myscript.sh`
5. `./myscript.sh` → run it

**Expected outcome:** Terminal prints both lines back to you.

## 10. Key Terms
| Term | Meaning |
|---|---|
| Script | A file of saved commands run automatically |
| Shebang | The `#!/bin/bash` line telling Linux which interpreter to use |
| Variable | A named container for storing a value |
| Loop | Code that repeats an action |
