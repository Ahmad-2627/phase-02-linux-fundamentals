# 🐧 Topic 2.1 — Why Hackers Use Linux

## 1. Topic Overview
Almost every penetration testing tool, every hacking distro (like Kali Linux), and every serious security professional works on Linux. Before touching any tool, you need to understand **why** — not just accept it as tradition.

## 2. Concept Explanation
Linux is a **free, open-source operating system**. "Open-source" means anyone can see, modify, and understand its internal code (unlike Windows, where Microsoft keeps the code secret/closed).

Hackers (ethical and malicious) prefer Linux because:
- **Full control** — you can access and modify almost every part of the system.
- **Free and customizable** — no license cost, and you can strip it down or build it up however you want.
- **Built for the terminal** — most powerful hacking tools are command-line based, and Linux is designed around the terminal (text-based control), not just clicking buttons.
- **Same OS that servers use** — most of the internet (web servers, cloud servers) runs on Linux. To attack or defend Linux servers, you must understand Linux from the inside.

## 3. Internal Working
Windows hides a lot of what's happening — processes, permissions, files — behind graphical menus. Linux exposes the raw system to you:
- Every setting is usually a text file you can open and read.
- Every action can be done with a command, and you can see exactly what that command does.
- The system doesn't stop you from making changes (even risky ones) — it trusts you know what you're doing.

This transparency is exactly what a security professional needs: the ability to see and control everything, not just what a menu allows.

## 4. Why It Matters
As a pentester, you will:
- Run Kali Linux (a hacking-focused Linux distro) as your main attack machine.
- Attack Linux-based servers (a huge percentage of real-world targets).
- Write and run scripts that only work properly in a Linux/terminal environment.

If you don't understand Linux itself, you'll only be copy-pasting commands without knowing why they work — dangerous in professional work.

## 5. Real-World Example
A company's web server runs on **Ubuntu Linux** (a popular Linux distro). During a penetration test, you gain limited access to that server. To escalate your access (get more control) and explore the system, you need to understand:
- Linux file structure
- Linux permissions
- Linux processes

Without this knowledge, even with access, you'd be stuck.

## 6. Attacker Perspective
Attackers use Linux because:
- Hacking tools (Nmap, Metasploit, Burp Suite, etc.) are built and tested primarily on Linux first.
- Linux lets you script and automate attacks easily using Bash (Linux's command-line scripting language).
- Many real-world targets (servers, IoT devices, cloud systems) run Linux — so understanding it means understanding your target.

## 7. Defender Perspective
Defenders (blue team — the security team protecting systems) also need Linux knowledge to:
- Secure Linux servers properly (most company servers run Linux).
- Read Linux logs (records of system activity) to detect attacks.
- Harden (strengthen) Linux systems against the very techniques attackers use.

## 8. Common Mistakes Beginners Make
- Thinking Linux is "just for hackers" — it's actually the backbone of the internet (web servers, cloud, IoT).
- Jumping straight into tools without learning basic Linux commands first.
- Being afraid of the terminal (black screen with text) instead of practicing it daily.

## 10. Key Terms
| Term | Simple Meaning |
|---|---|
| Open-source | Code anyone can view/modify, unlike closed systems like Windows |
| Distro (distribution) | A specific version/flavor of Linux (e.g., Kali, Ubuntu) |
| Terminal | Text-based tool to control the computer using typed commands |
| Bash | The scripting/command language used in Linux terminals |
