# 🧭 Topic 2.4 — Navigation Commands

## Picture this
Imagine Kali's file system as a huge building with thousands of rooms (folders) and files inside each room. Right now you're standing in one room with no map. Navigation commands are how you move through that building, see what's around you, and find exactly what you need — all without ever touching a mouse.

## 1–2. Concept
Linux organizes everything in a **tree structure** starting from one root folder called `/`. Every file and folder branches from there. You move through it using a few core commands:

| Command | Meaning |
|---|---|
| `pwd` | Print Working Directory — shows where you currently are |
| `ls` | List — shows files/folders in current location |
| `cd foldername` | Change Directory — move into a folder |
| `cd ..` | Move one folder up (back out) |
| `cd /` | Jump straight to root (top of the tree) |
| `cd ~` | Jump to your home folder |

## 3. Internal Working
Every folder is just an entry in the file system that points to other files/folders. When you type `cd`, Bash simply updates its memory of "where you currently are" — it doesn't move any files, it moves *you* (your active position).

## 4. Why It Matters
Every hacking tool you'll use expects you to navigate to specific folders — tool directories, target files, output logs. If you can't move around confidently, you can't use any tool efficiently.

## 5. Real-World Example
During a real pentest, after gaining access to a target's server, the very first thing you do is navigate around (`pwd`, `ls`, `cd`) to explore what files, folders, and data exist — this is part of **post-exploitation** (what happens after you get in).

## 6. Attacker Perspective
Attackers navigate quietly and quickly to find valuable files (passwords, configs, source code) without wasting time — speed matters before getting detected.

## 7. Defender Perspective
Defenders monitor logs of folder access — unusual navigation patterns (e.g., someone browsing sensitive folders they shouldn't) is a red flag in security monitoring.

## 8. Common Mistakes
- Forgetting Linux is case-sensitive (`Desktop` ≠ `desktop`).
- Confusing `cd ..` (go up one level) with `cd ~` (jump home).

## 9. Practical Lab
**Objective:** Practice moving around the file system.

**Steps:**
1. `pwd` → note your current location
2. `ls` → see what's there
3. `cd Desktop` → move into Desktop folder
4. `ls` → see what's inside
5. `cd ..` → go back up
6. `cd /` → jump to root, then `ls`
7. `cd ~` → return home

**Expected outcome:** You can move to any folder and back without getting lost.

**Key learning point:** `pwd` is your "you are here" marker — use it whenever unsure.

## 10. Key Terms
| Term | Meaning |
|---|---|
| Root (`/`) | The very top of the Linux file system tree |
| Home directory (`~`) | Your personal default folder |
| Directory | Linux word for "folder" |
