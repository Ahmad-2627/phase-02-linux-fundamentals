# Phase 02 — Linux Fundamentals

Part of my structured ethical hacking roadmap.

## What This Covers
Core Linux skills every penetration tester needs: terminal usage,
file system navigation, permissions, user/group management,
processes and services, package management, SSH, and Bash scripting.

## Topics
- 2.1 Why Hackers Use Linux
- 2.2 Installing Kali Linux (VirtualBox)
- 2.3 Terminal Basics
- 2.4 Navigation Commands
- 2.5 File and Directory Management
- 2.6 File Permissions (chmod, chown)
- 2.7 Users and Groups
- 2.8 Processes and Services
- 2.9 Package Management (apt)
- 2.10 SSH — Remote Access
- 2.11 Bash Scripting Basics

## Status
Completed — July 2026

## Key Takeaways
Learned why Linux is the standard OS for penetration testing:
open-source transparency and full system control. Practiced
the file permission model in depth, including the difference
between chmod 777 (open access) and SUID (identity-based
execution) — a distinction that matters directly for privilege
escalation work later in the roadmap.
